# Real interval clock updated #3754

## Main change

Real interval clock is now similar to rational interval clock by directly computing the coming interval, not the interval after that, https://github.com/modelica/ModelicaSpecification/pull/3754

## Reason

Consistency, user-friendliness, and that tools already used this semantics.

## Breaks

If a tool had implemented the previous semantics the rational interval clock would have used `previous(nextInterval)` for the next interval, not `nextInterval` in the example below.

``` modelica
  Real nextInterval(start = 0.002);
  Real y(start = 0);
equation
  when Clock(nextInterval) then
    // Real interval clock that after change ticks at 0, 0.003 (computed at 0s), 0.007, 0.012, ...

    // Previously the specification stated it ticked at 0, 0.002 (based on start-value), 0.005 (computed at 0s), ...
    nextInterval = previous(nextInterval) + 1e-3;
    y = previous(y) + 1;
  end when;
```
